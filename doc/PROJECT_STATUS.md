## 🐛 알려진 이슈 및 버그

### 해결됨 ✅
#### Issue #1: API 경로 이중 호출 문제
- **심각도:** 🟠 High
- **설명:** Frontend에서 `/api/api/...` 형태로 이중 경로 호출
- **원인:** `baseURL` 설정 누락 및 서비스 파일에서 `/api` 하드코딩
- **해결:** `api.js`에 `baseURL: '/api'` 설정, 서비스 파일에서 상대 경로 사용
- **상태:** ✅ 해결됨 (2026-01-10)

#### Issue #2: PostgreSQL 예약어 충돌 (user 테이블)
- **심각도:** 🔴 Critical
- **설명:** User Entity에서 `user` 테이블명 사용으로 SQL 문법 오류
- **해결:** `@Table(name="users")` 어노테이션 추가
- **상태:** ✅ 해결됨

#### Issue #3: Vite Proxy IPv6 연결 실패
- **심각도:** 🟠 High
- **설명:** `localhost`를 IPv6(`::1`)로 해석하여 ECONNREFUSED 오류
- **해결:** Proxy 타겟을 `http://127.0.0.1:8080`으로 명시적 변경
- **상태:** ✅ 해결됨

#### Issue #4: window.confirm 브라우저 무응답
- **심각도:** 🟡 Medium
- **설명:** 배너/이벤트 삭제 시 네이티브 confirm 다이얼로그 무응답
- **해결:** 커스텀 확인 모달 컴포넌트 구현
- **상태:** ✅ 해결됨 (2026-01-11)

#### Issue #5: 롤링 배너 이미지 404 오류
- **심각도:** 🟡 Medium
- **설명:** `/uploads/seed/` 경로 이미지 로딩 실패
- **원인:** Static 리소스 설정 누락
- **해결:** Spring Boot `WebConfig`에 리소스 핸들러 추가
- **상태:** ✅ 해결됨 (2026-01-11)

### 현재 이슈 없음
현재 시스템에 심각한 버그는 없으며, 미완료 기능은 구현 예정 항목입니다.

### 2. 성능 최적화
**현재 상태:** 기본 구현만 완료
**개선 필요:**
- 게시물 목록 조회 시 N+1 쿼리 문제 (Lazy Loading → Eager Loading)
- 이미지 썸네일 자동 생성 (원본 이미지 리사이징)
- Redis 캐싱 도입 (인기 게시물, 해시태그 통계)

### 3. 에러 처리 표준화
**현재 상태:** 개별 컨트롤러마다 에러 처리
**개선 필요:**
- GlobalExceptionHandler 확장
- 표준 에러 응답 포맷 정의
  ```json
  {
    "timestamp": "2026-01-11T19:25:00",
    "status": 404,
    "error": "NOT_FOUND",
    "message": "게시물을 찾을 수 없습니다.",
    "path": "/api/articles/999"
  }
  ```

### 4. 로깅 개선
**현재 상태:** 기본 Spring Boot 로깅
**개선 필요:**
- Logback 설정 커스터마이징
- 파일 로깅 (일별 로테이션)
- 에러 로그 별도 파일 저장
- ELK Stack 연동 (선택사항)

### 5. 코드 리팩토링
**현재 상태:** MVP 빠른 구현으로 일부 중복 코드 존재
**개선 필요:**
